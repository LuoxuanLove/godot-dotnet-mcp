# 用户扩展

用户扩展（User 工具）是插件在授权边界内的能力增长机制。你可以在 `custom_tools/` 中放置 `.gd` 脚本来增加新的 MCP 工具，无需修改插件本体，无需重建。

管理入口：`plugin_evolution` 工具组（7 个工具），以及 `MCPDock > Tools` 页中的 User 分类。

---

## 加载机制

`intelligence/executor.gd` 在 `_init()` 时自动扫描 `custom_tools/` 目录，对每个 `.gd` 文件执行三步校验：

1. **可实例化**：脚本能够正确加载并实例化
2. **实现接口**：具备 `handles`、`get_tools`、`execute` 三个方法
3. **前缀合规**：`get_tools()` 返回的所有工具名以 `user_` 开头

三步全部通过后，脚本以 `set_meta("is_user_tool", true)` 标记并加入 Intelligence 工具列表。任一步骤失败都会跳过该文件并记录警告。

**加载路径**：`res://addons/godot_dotnet_mcp/custom_tools/`

---

## 工具接口规范

```gdscript
@tool
extends RefCounted

func handles(tool_name: String) -> bool:
    return tool_name == "user_my_tool"

func get_tools() -> Array:
    return [
        {
            "name": "user_my_tool",          # 必须以 user_ 开头
            "description": "工具说明",
            "inputSchema": {
                "type": "object",
                "properties": {
                    "param": {"type": "string"}
                },
                "required": ["param"]
            }
        }
    ]

func execute(tool_name: String, args: Dictionary) -> Dictionary:
    if tool_name == "user_my_tool":
        var param = str(args.get("param", ""))
        return {"success": true, "result": "done: %s" % param}
    return {"success": false, "error": "Unknown tool: %s" % tool_name}
```

**注意**：

- 若同一脚本中任意一个工具名不以 `user_` 开头，整个文件被跳过
- 单个脚本可以声明多个工具，只需 `handles` 和 `get_tools` / `execute` 对应即可
- 脚本文件需放在 `custom_tools/` 目录根部（不支持子目录扫描）

## Dock 可见性

User 工具默认不在 Dock Tools 页中显示。通过以下方式开启：

- MCP 工具：`plugin_developer_user_visibility` 传入 `enabled: true`
- Dock 操作：`MCPDock > Tools > 设置 > 显示 User 工具`

开启后，User 分类会出现在工具树的独立根分组中。

## plugin_evolution 工具参考

以下 7 个工具负责 User 工具的完整生命周期管理（均以 `plugin_evolution_` 前缀调用）：

| 工具名 | 功能 | 关键参数 |
|---|---|---|
| `list_user_tools` | 列出所有已注册 User 工具 | — |
| `scaffold_user_tool` | 预览或创建 User 工具脚本框架 | `tool_name`（必填）、`authorized`（两步确认）、`display_name`、`description` |
| `delete_user_tool` | 预览或删除指定 User 工具脚本 | `script_path`（必填）、`authorized`、`agent_hint` |
| `restore_user_tool` | 预览或恢复最近一次被删除的 User 工具 | `authorized`、`agent_hint` |
| `user_tool_audit` | 读取最近的用户工具审计记录 | `limit`、`filter_action`、`filter_session` |
| `check_compatibility` | 对比现有 User 工具与当前脚手架版本的兼容性 | — |
| `usage_guide` | 返回推荐的授权工作流与使用说明 | — |

---

## 两步授权流程

`scaffold_user_tool`、`delete_user_tool`、`restore_user_tool` 均采用两步确认：

**第一步（预演）**：不传 `authorized` 或传 `authorized: false`，返回将要执行的操作描述，不实际写入。

**第二步（执行）**：确认预演结果后，传 `authorized: true` 正式执行。

```
# 预演
plugin_evolution_scaffold_user_tool(tool_name="user_my_tool")
# → 返回脚手架预览

# 执行
plugin_evolution_scaffold_user_tool(tool_name="user_my_tool", authorized=true)
# → 写入 custom_tools/user_my_tool.gd
```

---

## backup 机制

删除 User 工具时，脚本文件自动备份至 `custom_tools/.backup/`，不会直接丢失。

使用 `restore_user_tool` 可恢复最近一次备份的脚本。

---

## 审计记录

所有 User 工具操作（创建、删除、恢复）均写入审计记录：

**路径**：`user://godot_dotnet_mcp_user_tool_audit.json`

通过 `user_tool_audit` 工具读取：

```
plugin_evolution_user_tool_audit(limit=20, filter_action="scaffold")
```

可按 `filter_action`（scaffold / delete / restore）或 `filter_session` 过滤。
