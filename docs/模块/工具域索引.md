#### system 域详细说明

**目录结构**：

```
tools/system/
├── executor.gd        # 调度器
├── atomic_bridge.gd   # 原子工具调用桥，含写保护
├── impl_project.gd    # 项目级工具
├── impl_scene.gd      # 场景级工具
├── impl_script.gd     # 脚本级工具
├── impl_index.gd      # 索引与搜索工具
└── lsp_client.gd      # Godot LSP 客户端（StreamPeerTCP，供 impl_script 调用）
```

**15 个内置工具**：

| 工具名 | 所属 impl | 说明 |
|---|---|---|

## 兼容 wrapper

`tools/` 根目录保留了一批 `*_tools.gd` 兼容或辅助脚本：

- `scene_tools.gd`、`node_tools.gd`、`script_tools.gd`
- `resource_tools.gd`、`project_tools.gd`
- `plugin_shared.gd`（Plugin 域 executor 的共享基类）

新能力应优先收敛到 `tools/<category>/executor.gd`，不应继续膨胀 wrapper。

## 文档维护约束

- 新增 executor 目录时，必须同步更新本索引。
- 若把某个兼容 wrapper 废弃或迁移到 executor，必须在此标明。
- 若新增 `plugin_*` 工具或 `custom_tools/` 工作流变化，也必须同步更新本索引和 [用户扩展.md](用户扩展.md)。
