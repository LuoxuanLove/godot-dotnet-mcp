### Core

| Category | 主要职责 | 代表工具 |

| `node` | 节点查询、属性、变换、生命周期与处理开关 | `node_query`、`node_property`、`node_transform`、`node_lifecycle` |

| `resource` | 资源查询、创建、文件操作与贴图赋值 | `resource_query`、`resource_create`、`resource_file_ops`、`resource_texture` |
| `filesystem` | 目录、文件、JSON 与全文搜索 | `filesystem_directory`、`filesystem_file_read`、`filesystem_json`、`filesystem_search` |

## 兼容 wrapper

除了 executor 目录，`tools/` 根目录还保留了一批 `*_tools.gd` 兼容或辅助脚本，例如：

- `scene_tools.gd`
- `node_tools.gd`
- `script_tools.gd`
- `resource_tools.gd`
- `project_tools.gd`
- `plugin_shared.gd`

这些脚本的用途通常是：

- 封装旧接口
- 抽公共 helper
- 降低 executor 内重复代码

新能力应优先收敛到 `tools/<category>/executor.gd`，而不是继续膨胀 wrapper。

## 文档维护约束

- 新增 executor 目录时，必须同步更新本索引。
- 若把某个兼容 wrapper 废弃或迁移到 executor，必须在此标明。
- 若新增 `plugin_*` 工具或 `custom_tools/` 工作流变化，也必须同步更新本索引和 [自进化系统.md](自进化系统.md)。
