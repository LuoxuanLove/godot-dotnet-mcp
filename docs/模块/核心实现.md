# 核心实现

本文档汇总 `localization/`、`plugin/config/`、`plugin/runtime/`、`custom_tools/` 和共享辅助脚本的职责边界，作为“除工具域本体外”的核心实现说明。

---

## 目录分层

```text
localization/
plugin/
  ├─ config/
  └─ runtime/
tools/
  ├─ core/
  ├─ *_tools.gd
  └─ <category>/executor.gd
custom_tools/
```

---

## `localization/`

### 组成

- `localization_service.gd`
- `locale_en.gd`
- `locale_zh_cn.gd`
- `locale_zh_tw.gd`
- `locale_ja.gd`
- `locale_de.gd`
- `locale_es.gd`
- `locale_fr.gd`
- `locale_pt.gd`
- `locale_ru.gd`

### 职责

- 加载所有语言字典
- 切换当前语言
- 提供 `get_text()` 与语言显示名辅助
- 为 Dock UI 和工具 profile 文案供值

### 约束

- 语言文件键集合应与 `locale_en.gd` 对齐
- 场景文件中的固定文案不应作为最终显示来源
- 插件内部根控件普遍关闭 `auto_translate_mode`

---

## `plugin/config/`

### `config_paths.gd`

负责：

- 桌面端客户端配置路径
- MCP URL 配置模板
- Claude Code / Codex 命令模板

### `client_config_service.gd`

负责：

- 调用 `config_paths.gd`
- 封装桌面端配置写入
- 做最小合并写盘

### `settings_store.gd`

负责：

- 插件设置文件
- 自定义 profile 文件
- profile 重命名 / 删除
- 工具配置 JSON 导入导出

## `plugin/runtime/`

这一层是当前插件实现最核心的服务层。

### 状态与生命周期

- `plugin_runtime_state.gd`
- `server_runtime_controller.gd`
- `plugin_reload_coordinator.gd`

### 诊断与调试

- `plugin_self_diagnostic_store.gd`
- `mcp_runtime_debug_store.gd`
- `mcp_editor_debugger_bridge.gd`：注册到 `EditorDebuggerPlugin`，接收主项目调试会话事件，转发至 `MCPDebugBuffer` 和 `MCPRuntimeDebugStore`
- `mcp_runtime_bridge.gd`：以 Autoload 形式安装到主项目，主项目运行时可通过 `emit_event` / `emit_log` / `emit_info` / `emit_warning` / `emit_error` 发送结构化事件；双通道（`godot_mcp/runtime_event` + `godot_mcp/runtime_log`），最多缓存 300 条，EngineDebugger 不活跃时 fallback 写入 `user://godot_mcp_runtime_bridge_events.json`

### 服务与目录

- `mcp_http_server.gd`
- `tool_catalog_service.gd`
- `user_tool_service.gd`

这些对象共同支撑：

- 设置与 profile 投影
- 内嵌 server 生命周期
- 工具目录与权限裁剪
- User 工具发现与管理
- 自诊断链路

## `tools/` 共享层

除了各 category executor 外，`tools/` 目录还包含一批共享脚本：

### 基础与工具兼容层

- `base_tools.gd`
- `plugin_shared.gd`
- `scene_tools.gd`
- `node_tools.gd`
- `script_tools.gd`
- 其它 `*_tools.gd`

### 通用辅助

- `mcp_debug_buffer.gd`
- `mcp_file_utils.gd`
- `mcp_json_utils.gd`
- `mcp_node_utils.gd`
- `mcp_script_parser.gd`
- `mcp_type_utils.gd`

### 核心

- `tool_registry.gd`
- `core/tool_loader.gd`

这些脚本的关系是：

- `tool_registry.gd` 定义“有什么”
- `tool_loader.gd` 负责“怎么装、怎么调”
- `base_tools.gd` 和其它 helper 负责“怎么少重复代码”

---

## `custom_tools/`

这个目录承载用户通过 `plugin_evolution` 工具或脚手架维护的 User 工具脚本。当前约束是：

- 不写入内置工具域目录
- 通过 `plugin_evolution` 两步授权流程创建或删除
- 由 `system/executor.gd._load_custom_tools()` 扫描发现
- 删除时可写入 `.backup/` 以支持恢复

---

## 核心设计取向

当前插件实现强调四个原则：

- 薄入口：`plugin.gd` 只做装配和桥接
- 场景化 UI：Dock 页签单独成场景和控制器
- 域拆分工具：按 category executor 划分而不是单个大工具
- 结构化观测：错误、操作、会话与日志尽量结构化记录
