# 安装与发布

本文档说明插件包与 Bridge 包的安装方式、发布结构和生成策略。

---

## 安装方式

### 1. 直接复制目录

将插件目录放到目标 Godot 项目：

```text
addons/godot_dotnet_mcp
```

然后在 Godot 中：

1. 打开项目。
2. 进入 `Project Settings > Plugins`。
3. 启用 `Godot .NET MCP`。
4. 打开右侧 `MCPDock`。
5. 使用 `Install Bridge` 或状态按钮完成 Bridge 安装。

### 2. Git Submodule

当前仓库根目录不再是插件本体，插件位于仓库内的 `addons/godot_dotnet_mcp/` 子路径。推荐克隆至临时目录再复制：

```bash
git clone --depth=1 https://github.com/LuoxuanLove/godot-dotnet-mcp.git _godot-dotnet-mcp
cp -r _godot-dotnet-mcp/addons/godot_dotnet_mcp addons/godot_dotnet_mcp
rm -rf _godot-dotnet-mcp
```

若需以子模块方式跟踪，需在 sparse checkout 模式下仅检出 `addons/godot_dotnet_mcp/` 路径。

### 3. 发布包

GitHub Releases 中建议同时提供两个独立附件：

- 插件包：`godot-dotnet-mcp-0.6.0.zip`
- Bridge 包：`godot-dotnet-mcp-bridge-win-x64-0.6.0.zip`

插件包保持如下结构：

```text
addons/
└── godot_dotnet_mcp/
```

Bridge 包建议以发布后的 `win-x64` self-contained 单文件 exe 为核心，附带校验文件和最小安装说明。

---

## Bridge 发布策略

v0.6 的 Bridge 发布目标是让用户无需安装额外的 .NET runtime。

要求如下：

- 源码编译目标保持 `net8.0`
- 发布必须使用 `self-contained`
- 发布优先使用单文件输出
- 首发仅保证 `win-x64`
- Bridge exe 必须可以脱离系统运行时直接启动

这意味着：

- 用户安装插件时，不会被要求安装 .NET
- 用户安装 Bridge exe 时，也不会被要求安装 .NET
- 安装器和发布包只处理版本校验、解压和路径注册
- 发布机需要能获取对应的 `win-x64` runtime pack 缓存或访问源，但这不传递给最终用户

---

## 启动后检查

建议按以下顺序确认安装成功：

1. `Project Settings > Plugins` 中插件状态为启用
2. `MCPDock` 已出现在编辑器右侧
3. Bridge 版本与插件版本一致
4. Bridge 健康检查返回 `status = ok`
5. MCP `tools/list` 返回工具列表

### 插件包

对外分发时建议包含：

- `addons/godot_dotnet_mcp/plugin.cfg`
- `addons/godot_dotnet_mcp/plugin.gd`
- `addons/godot_dotnet_mcp/plugin/runtime/mcp_http_server.gd`
- `addons/godot_dotnet_mcp/tools/`
- `addons/godot_dotnet_mcp/ui/`
- `addons/godot_dotnet_mcp/localization/`
- `addons/godot_dotnet_mcp/docs/`
- `addons/godot_dotnet_mcp/README.md`
- `addons/godot_dotnet_mcp/LICENSE`

### Bridge 包

Bridge 包建议包含：

- 发布后的 `GodotDotnetMcp.DotnetBridge.exe`
- `SHA256SUMS.txt`
- 最小 `README.txt` 或安装说明

## 发布包生成方法

### 插件包

推荐从插件根目录生成发布包，并保持 zip 内部结构为 `addons/godot_dotnet_mcp/`。

标准步骤：

1. 删除旧的 `release/` 发布包和旧校验值。
2. 从 `addons/godot_dotnet_mcp/` 复制发布内容到临时打包目录。
3. 排除内部维护目录。
4. 生成插件 zip。
5. 重新计算 `SHA256SUMS.txt`。

### Bridge 包

建议从 `addons/godot_dotnet_mcp/dotnet_bridge/` 执行：

```bash
dotnet publish DotnetBridge.csproj -c Release -p:PublishProfile=WinX64SelfContained
```

然后把 `publish/` 输出目录打包成 Bridge 发行包。
