# 安装与发布

本文档描述 v1.0 重构期间已经确定的安装、打包和发布口径。

当前统一原则是：

- `Central Server` 是唯一对外公开的 MCP 入口。
- Godot 插件负责编辑器侧附着、实时工具执行和本地诊断。
- 旧 `Dotnet Bridge` 产品壳已经移除，不再作为正式产品主入口。

---

## 产品边界

### 对外产品

- `central-server-win-x64`
  独立安装的中心服务，负责 `workspace_*`、`dotnet_*`，并在需要时调度 `system_*`。
- `plugin-lean`
  仅包含 Godot 插件，适合已经独立安装中心服务的环境。
- `plugin-bundled-win-x64`
  插件加本地 bundled 引导包，适合作为中心服务未预装时的兜底方案。

### 非正式主入口

- 插件内嵌 HTTP transport
  当前仍在过渡期存在，但长期定位是 Host 与插件之间的内部通道，而不是对外产品入口。

---

## 安装模式

### 1. 独立安装中心服务

这是 v1.0 的主路径。

适用场景：

- 需要把 MCP 客户端统一接到一个外部宿主。
- 希望在未打开 Godot 时也能直接使用 `workspace_*`、`dotnet_*`。
- 需要按项目管理、按需拉起 Godot。

典型流程：

1. 安装 `central-server-win-x64`。
2. 把 MCP 客户端配置为启动 `GodotDotnetMcp.CentralServer --stdio`。
3. 在 Godot 项目中安装并启用插件。
4. 由中心服务在需要 `system_*` 时附着或拉起 Godot。

### 2. 插件 bundled 本地引导

这是兜底路径，不是源码开发主路径。

适用场景：

- 用户尚未独立安装中心服务。
- 希望插件在本地帮助完成中心服务引导安装。

约束：

- bundled 包只在发布阶段注入。
- 不再把中心服务 zip 长期跟踪在源码仓中。
- bundled 内容属于发布产物，不属于源码输入。

### 3. 源码开发模式

适用于仓库开发和调试。

做法：

1. 从源码复制 `addons/godot_dotnet_mcp/` 到 Godot 项目。
2. 使用 `dotnet run --project central_server/CentralServer.csproj -- --health` 验证宿主。
3. 在 Godot 中启用插件并观察 `MCPDock > Server`。

---

## 发布结构

当前发布脚本 `scripts/publish_central_server.ps1` 负责把产物输出到：

```text
dist/
  central-server-win-x64/
  plugin-lean/
  plugin-bundled-win-x64/
```

含义如下：

- `dist/central-server-win-x64/`
  独立中心服务发布目录。
- `dist/plugin-lean/`
  不带 bundled 中心服务包的插件目录。
- `dist/plugin-bundled-win-x64/`
  由脚本在发布阶段注入中心服务 zip/hash 的插件目录。

---

## 源码仓边界

### 已明确禁止

- 把 `release_dist/`、`dist/` 重新当作源码输入。
- 把 bundled 中心服务 `zip`、`sha256` 再次提交回 `addons/godot_dotnet_mcp/central_server_packages/`。
- 把临时下载缓存、日志、解压目录长期纳入版本控制。

### 当前护栏

- `.gitignore` 已收口 `dist/`、`release_dist/`、`tmp/` 和本地调试输出。
- `addons/godot_dotnet_mcp/central_server_packages/.gitignore` 只保留说明文件和忽略规则。
- `scripts/validate_refactor_guardrails.ps1` 会阻止 bundled 制品和构建输出重新被 git 跟踪。

---

## 安全约束

安装与打包链必须遵守以下规则：

- 强制覆盖插件时必须保留用户 `custom_tools/`。
- 如果保留下载与更新能力，必须补齐哈希校验、来源限制和安全解压。
- 失败时必须返回结构化错误，不能静默回退为不确定状态。

当前已完成：

- `Central Server --install-plugin` 已接管主安装入口。
- `PluginInstaller` 在 `--force` 覆盖安装时会保留并恢复 `custom_tools/`。

## 验证建议

### 安装闭环

建议按以下顺序验证：

1. `Central Server --health` 返回正常。
2. 插件在 Godot 中可启用，`MCPDock` 可打开。
3. `MCPDock > Server` 能正确显示本地中心服务状态。
4. 独立安装模式下，客户端能通过 `GodotDotnetMcp.CentralServer --stdio` 接入。
5. bundled 模式下，插件能完成本地引导安装。

### 发布闭环

建议至少检查：

1. `dist/central-server-win-x64/` 存在并可运行。
2. `dist/plugin-lean/` 不包含 bundled zip。
3. `dist/plugin-bundled-win-x64/` 包含发布阶段注入的 bundled zip/hash。
4. 源码树中没有重新出现被跟踪的 bundled 制品。

## 兼容说明

- 新的安装说明、发布说明和外部接入文档都必须以 `Central Server` 为准。
- 插件侧仍有过渡中的内部 transport，但它们不再代表公开产品入口。
- 只有在你明确确认重构结束后，临时计划文档和剩余过渡叙事才会最终收口。
