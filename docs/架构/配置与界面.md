# 配置与界面

## Dock 结构

当前 Dock 聚焦三类任务：

- `Server`：服务状态、端口、语言、调试开关
- `Tools`：工具分类启停与统计
- `Config`：面向外部 MCP 客户端的连接配置展示与写入

不再承担：

- 作者展示
- 宣传信息
- 与工具能力无关的说明区块

## 入口与页签职责

Dock 入口仍由 `plugin.gd` 注册到 Godot 编辑器右侧面板，但节点结构已改为场景化：

- `ui/mcp_dock.tscn`：Dock 根场景，负责标题栏、状态灯和 TabContainer
- `ui/server_panel.tscn`：服务状态与基础设置
- `ui/tools_tab.tscn`：工具 profile、domain/category/tool 启停
- `ui/config_panel.tscn`：客户端配置与命令生成

对应控制器脚本只接收 `apply_model(model)`，不直接持有服务对象。

## `Server` 页签操作说明

### 可见信息

- 状态：当前服务是否运行
- 端点地址：当前 host/port 组合出的 MCP 地址
- 活跃连接：当前连接中的客户端数量
- 总请求数：累计请求数和累计连接数
- 最近请求：最后一次 HTTP 方法与时间

### 可执行操作

1. 修改端口。
2. 切换自动启动。
3. 切换调试日志。
4. 切换插件语言。
5. 启动服务。
6. 重启服务。
7. 停止服务。
8. 完全重载插件。

### 行为说明

- “启动”会调用 `plugin.gd -> ServerRuntimeController.start()`。
- “重启”会先 `reinitialize()` 再 `start()`，用于刷新端口、调试模式和禁用工具列表。
- “停止”只停止 `MCPServer` 监听，不销毁 Dock。
- “完全重载插件”不再只是运行时 `detach()` / `attach()`，而是由独立协调器节点调用 Godot 插件层的 `set_plugin_enabled(false/true)`，尽量贴近项目设置中关闭再启用插件的效果。
- 完整重载前会记录 `MCPDock` 当前外层标签、内部页签和页签内焦点快照；新实例启动后先切回宿主 Dock 的 `MCPDock` 标签，再恢复内部页签与原焦点控件。
- 语言切换只更新 `settings.language` 并触发轻量 `apply_model()` 刷新，不再重建整个 Dock。
- 语言切换与完整重载后的焦点恢复都遵循“只恢复原控件，不强制抢焦点”的策略；如果原控件不存在或不可聚焦，则保持静默。
- 语言菜单统一显示为“自称（当前语言译名）”，例如 `English（英文）`、`日本語（日语）`。

## `Tools` 页签操作说明

### 可执行操作

1. 选择内置 profile：`slim`、`default`、`full`。
2. 保存当前启停状态为自定义 profile。
3. 按 domain 折叠或展开。
4. 按 category 折叠或展开。
5. 启停单个工具、整个分类或整个 domain。
6. 全部展开或全部折叠。

### 行为说明

- profile 切换会直接重算 `disabled_tools`，并同步到 `MCPServer`。
- 自定义 profile 会落盘到 `user://godot_dotnet_mcp_tool_profiles/*.json`。
- 如果当前 `disabled_tools` 与选中 profile 不一致，说明文案会追加“已修改”状态提示。
- 工具树的启用数使用运行中服务返回的真实工具清单计算，而不是静态写死。
- 工具树展示固定按 `domain -> category -> tool` 三层缩进，缩进量随 `editor_scale` 同步缩放，只影响视觉结构，不影响 profile 或启停逻辑。

## `Config` 页签操作说明

配置生成逻辑已从 UI 脚本中抽离，统一由辅助模块处理。这样做的目的有两个：

- 降低 Dock 代码与字符串模板的耦合
- 让不同客户端配置的变更不直接污染 UI 层

### 平台筛选

配置页顶部先选择 Agent 平台，然后只展示该平台对应的配置内容，不再把所有平台信息同时堆叠在一个长列表里。

当前平台分为两类：

- 配置文件类：Claude Desktop、Cursor、Gemini
- CLI 命令类：Claude Code、Codex

### 推荐使用流程

建议按以下顺序操作配置页：

1. 先在顶部平台下拉中选中目标客户端。
2. 确认页面当前显示的是“配置文件区”还是“CLI 配置区”。
3. 如为桌面端客户端，先核对配置文件路径和 JSON 内容，再决定使用“写入配置”或“复制”。
4. 如为 CLI 客户端，先确认命令文本；若为 `Claude Code`，再额外确认 `user / project` 作用域是否正确。
5. 完成复制或写入后，再回到客户端侧执行连接验证。

### 配置文件区

当前支持的一键写入客户端包括：

- Claude Desktop
- Cursor
- Gemini

每一项都包含：

- 配置文件路径
- 当前要写入的 JSON 内容
- 一键写入按钮
- 复制按钮

一键写入的行为是：

1. 先解析当前 JSON 文本。
2. 如果目标文件已存在且是合法 JSON，对其做最小合并。
3. 仅更新 `mcpServers["godot-mcp"]` 节点，避免覆盖文件中的其它服务器配置。
4. 目录不存在时先递归创建，再写盘。

补充说明：

- “写入配置”只更新 `mcpServers["godot-mcp"]`，不会清空目标文件里的其它 MCP 服务。
- “复制”复制的是当前文本框里的完整配置内容，适合手动粘贴到受版本管理或受限环境中的配置文件。

### CLI 配置区

当前支持的 CLI 平台包括：

- Claude Code
- Codex

其中：

- Claude Code 额外显示作用域下拉（`user` / `project`）
- Codex 只展示当前命令文本与复制操作

CLI 区域的行为约束如下：

- `作用域` 只影响 `Claude Code` 命令参数，不影响其它客户端。
- CLI 区域不提供“一键写入”，因为实际写入由外部命令完成。
- 当平台切到桌面端客户端时，CLI 区域会被隐藏，避免同时看到无关命令。

## 设置持久化

编辑器内设置应满足：

- 有默认值
- 可重置
- 重启后可恢复
- 与当前项目或当前编辑器的边界清晰

当前持久化路径如下：

| 路径 | 作用 |
|---|---|
| `user://godot_dotnet_mcp_settings.json` | 保存端口、语言、自动启动、调试模式、禁用工具、折叠状态，以及完整重载时的临时焦点快照 |
| `user://godot_dotnet_mcp_tool_profiles/` | 保存自定义工具 profile |

加载逻辑位于 `plugin/config/settings_store.gd`：

- 首次无设置文件时，默认折叠全部 category 与 domain
- 已有设置文件时，缺失字段会做兼容补齐
- `tool_profile_id` 为空时强制回退到 `default`

## 文档上的界面约束

当 UI 或设置项发生变化时，文档需要同时说明：

- 入口页签位置
- 设置键名或配置字段名
- 是否影响协议层行为
- 是否需要迁移已有客户端配置

## 实现方法

### UI 装配

当前实现链路如下：

1. `plugin.gd._build_dock_model()` 汇总状态、工具清单、配置文本和 `editor_scale`。
2. `ui/mcp_dock.gd` 将 model 分发到三个页签。
3. `ui/server_tab.gd`、`ui/tools_tab.gd`、`ui/config_tab.gd` 各自填充文本、按钮状态和下拉框。
4. 用户操作通过显式信号回到 `plugin.gd`，再调用服务对象或设置存储对象。

### UI 文本本地化

当前 Dock 根容器、Server / Tools / Config 三个页签、配置客户端标题和工具树名称都通过 localization key 填充，不再依赖场景文件中的硬编码文案。

为了避免 Godot 编辑器对常见英文词再次做自动翻译，Dock 根容器和三个页签根控件都显式关闭了 `auto_translate_mode`。因此插件内部语言切换完全由 `LocalizationService` 驱动，不依赖编辑器全局翻译状态。

本轮仅覆盖插件 UI 文本，不包含工具运行时错误信息的多语言化；后者仍留在后续阶段处理。

### 界面缩放

当前缩放策略以 `EditorInterface.get_editor_scale()` 为唯一倍率来源。缩放生效点包括：

- Dock 根容器最小尺寸
- 页签内边距
- 各类 `HBoxContainer` / `VBoxContainer` / `GridContainer` 的间距
- `SpinBox`、`OptionButton`、`Button` 的最小高度
- `TextEdit` 的最小高度
- 工具树动态生成说明文字的字号

实现原因：

- 原始常量在高 DPI 或编辑器缩放大于 `1.0` 时会显得过密
- 场景化重构后，旧版入口脚本里的缩放逻辑已经移除，必须在各自控制器内重建

### 为什么服务页和配置页需要额外布局约束

这两个页签包含：

- `GridContainer`
- 长文本 `Label`
- 只读 `TextEdit`
- 需要横向展开的 `OptionButton`

如果不显式补 `size_flags_horizontal = SIZE_EXPAND_FILL` 和随倍率调整的最小高度，会出现：

- 文本区域过窄
- 按钮高度小于编辑器整体视觉尺度
- 复选框文案贴边或被误判为未显示

## 排障建议

- 如果服务页显示状态正确但控件明显偏小，先检查 `editor_scale` 是否进入 model。
- 如果复选框只有勾选框没有文字，先检查 `server_tab.gd` 是否给 `AutoStartCheck` 和 `DebugCheck` 赋了 localization 文本。
- 如果配置页文本框不随宽度展开，先检查 `config_panel.tscn` 的 `Content`、各 section 和 `TextEdit` 是否都带有横向扩展约束。
- 如果一键写入失败，优先检查目标路径是否可写以及现有 JSON 是否可解析。
