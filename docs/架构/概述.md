# 架构概述

## 目标定位

`godot_mcp` 的目标不是覆盖 Godot 全部 API，而是作为一个在编辑器内运行、面向 Godot 4 项目的 MCP 服务层，提供稳定、可组合、可验证的自动化入口。

当前设计强调：

- 以编辑器内工具调用为中心，而不是离线脚本生成
- 以分类稳定为优先，而不是持续堆叠同义入口
- 以 Godot.NET 项目的静态分析与绑定检查为特色能力
- 保留必要的 GDScript 支持，但不再以 GDScript 为接口中心

## 分层结构

```text
EditorPlugin
  ├─ plugin.gd
  ├─ Dock Scene 装配
  ├─ 插件状态与设置装配
  ├─ 服务启停协调
  └─ 与 UI 控制器的显式信号连接

MCPServer
  ├─ plugin/runtime/mcp_http_server.gd
  ├─ tools/core/tool_loader.gd
  ├─ HTTP/TCP 监听
  ├─ JSON-RPC / MCP 请求分发
  ├─ tools/list
  └─ tools/call

Tool Executors
  ├─ tools/<category>/executor.gd
  ├─ tools/*.gd（兼容实现层）
  ├─ intelligence（高层工具：项目/场景/脚本/索引，含 Atomic Bridge）
  ├─ scene / node / script
  ├─ filesystem / resource / project
  ├─ animation / physics / navigation / audio / ui
  ├─ editor / debug / plugin 等域工具
  └─ custom_tools/*.gd（User 工具，经 intelligence/executor.gd 加载）

Shared Helpers
  ├─ base_tools.gd
  ├─ plugin/config/config_paths.gd
  └─ tools/core/*
```

## 运行时对象关系

当前插件入口已从单一超大脚本重构为薄装配层，运行时对象关系如下：

```text
plugin.gd
  ├─ PluginRuntimeState
  ├─ SettingsStore
  ├─ ServerRuntimeController
  ├─ ToolCatalogService
  ├─ ClientConfigService
  ├─ PluginReloadCoordinator
  ├─ MCPRuntimeBridge（Autoload，注入主项目运行时事件通道）
  ├─ MCPEditorDebuggerBridge（接收编辑器调试会话事件）
  └─ ui/mcp_dock.tscn
       ├─ ui/server_panel.tscn
       ├─ ui/tools_tab.tscn
       └─ ui/config_panel.tscn
```

这意味着：

- `plugin.gd` 只负责装配、状态同步和信号桥接
- Dock 页签不再直接创建服务对象，也不再直接读写配置文件
- 工具 profile 计算、配置文本生成、设置落盘都从 UI 中抽离
- 插件级“完整重载”由独立协调器节点跨实例完成，避免在当前插件实例销毁时丢失启用动作
- 未来扩展新页签时，优先新增独立场景和控制器，而不是继续膨胀入口脚本

## 核心职责

### `plugin.gd`

负责：

- 管理插件生命周期
- 装配 Dock 场景与控制器
- 连接状态、服务、工具目录与配置服务
- 对外暴露最小公共 API（`get_server` / `start_server` / `stop_server`）
- 负责把 `editor_scale`、当前语言、当前运行状态和配置文本投影到 UI model
- 负责保存与恢复完整重载前后的 Dock 页签 / 焦点快照

不负责：

- 具体 Dock 节点树构建
- 工具树 profile 匹配与计数计算
- 配置文件读写细节
- 协议级请求分发
- 具体按钮点击后的局部控件更新

### `plugin/runtime/mcp_http_server.gd`

负责：

- 接收请求
- 解析协议
- 构造工具列表
- 将工具调用转发给 `MCPToolLoader`
- 统一封装返回结构
- 维护连接数、请求数、最近请求方法和时间

### `tools/core/tool_loader.gd`

负责：

- 维护 `category -> script path -> runtime instance` 装载关系
- 扫描内置工具与 `custom_tools/*.gd`
- 预加载启用 category，保留其它 category 的定义缓存
- 提供单 domain / 全量热重载入口
- 汇总 load state、reload status 与性能摘要

### `tools/*.gd`

负责：

- 对某一能力域提供权威入口
- 在域内通过 `action` 分派子行为
- 维持输入、输出和错误结构的一致性
- 复用 `base_tools.gd` 的路径解析、节点序列化、JSON/Variant 归一化 helper

### `ui/*.tscn` + `ui/*.gd`

负责：

- 承载 Dock 根界面与各 Tab 的节点结构
- 通过显式信号向 `plugin.gd` 报告用户操作
- 只处理展示与交互，不直接管理服务生命周期
- 根据 `editor_scale` 调整间距、最小尺寸和文本区域高度
- 对插件内部文本关闭 Godot 自动翻译，统一由本地化服务驱动显示语言

### `plugin_*` 服务对象

负责：

- `plugin/runtime/plugin_runtime_state.gd`：运行态字段、默认设置、domain/profile 常量
- `plugin/config/settings_store.gd`：设置与自定义 profile 持久化
- `plugin/runtime/server_runtime_controller.gd`：`MCPHttpServer` 生命周期与状态代理
- `plugin/runtime/tool_catalog_service.gd`：domain/category/profile 计算
- `plugin/config/client_config_service.gd`：客户端配置文本与落盘逻辑

## 启动流程

1. `plugin.gd._enter_tree()` 调用 `_load_state()`，从 `user://godot_dotnet_mcp_settings.json` 和 `user://godot_dotnet_mcp_tool_profiles` 读取数据。
2. 插件设置语言并装配 `ServerRuntimeController`。
3. 安装 `MCPRuntimeBridge` Autoload 到当前 Godot 项目（在 `project.godot` 中注入）。
4. 注册 `MCPEditorDebuggerBridge` 到 `EditorDebuggerPlugin`，开始接收主项目调试会话事件。
5. 入口实例化 `ui/mcp_dock.tscn`，连接所有显式信号。
6. 如果是首次载入设置，按内置 profile 计算 `disabled_tools`。
7. 根据 `auto_start` 决定是否启动服务。
8. 如存在上一次完整重载留下的 Dock 焦点快照，则在首轮刷新后恢复宿主 Dock 标签、内部页签与原焦点。
9. `_process()` 每 0.5 秒刷新一次 Dock model，用于同步服务状态和统计。

## UI 数据流

当前 Dock 使用单向数据流：

1. `plugin.gd._build_dock_model()` 收集状态、统计、工具、配置文本和 `editor_scale`。
2. `mcp_dock.gd.apply_model()` 把 model 分发给三个 Tab。
3. Tab 控制器只把用户动作上抛为信号，例如 `start_requested`、`profile_selected`、`config_write_requested`。
4. `plugin.gd` 响应信号，更新状态、调用服务对象，再次 `_refresh_dock()`。

这个结构的直接收益：

- 不需要在 UI 脚本里保存服务引用
- 语言切换和缩放更新可以统一重渲染，其中语言切换保持为轻量 `apply_model()` 刷新
- 任何 Tab 的展示都可通过 model 回放，便于后续补测试或继续拆分

## 当前架构约束

- 工具名以分类稳定为先，不轻易引入重复前缀
- 返回值必须可被 MCP 客户端直接消费
- 写操作应尽量支持“写后读回验证”
- 场景、脚本、资源类工具优先服务编辑器内工作流
- category 对外命名保持稳定，内部目录结构允许继续演进
- 插件 UI 采用场景化组织，插件入口脚本保持薄装配层
- 页签脚本只处理展示和交互，不直接触碰 `plugin/runtime/mcp_http_server.gd`
- 界面缩放以 `EditorInterface.get_editor_scale()` 为唯一倍率来源
