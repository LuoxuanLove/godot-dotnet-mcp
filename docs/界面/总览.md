# 界面总览

本文档描述 `ui/` 目录下场景与脚本的关系，以及 Dock 根场景如何把统一 model 分发给 `Server`、`Tools`、`Config` 三个页签。

---

## UI 文件组成

当前 `ui/` 目录由以下文件组成：

| 路径 | 作用 |
|---|---|
| `ui/mcp_dock.tscn` | Dock 根场景 |
| `ui/mcp_dock.gd` | Dock 根控制器 |
| `ui/server_panel.tscn` | `Server` 页签场景 |
| `ui/server_tab.gd` | `Server` 页签控制器 |
| `ui/tools_tab.tscn` | `Tools` 页签场景 |
| `ui/tools_tab.gd` | `Tools` 页签控制器 |
| `ui/config_panel.tscn` | `Config` 页签场景 |
| `ui/config_tab.gd` | `Config` 页签控制器 |

---

## 根场景职责

`mcp_dock.tscn` 负责：

- 标题栏
- 状态灯
- 当前运行状态文本
- `TabContainer`
- 页签场景实例化承载位

`mcp_dock.gd` 负责：

- 实例化三个页签场景
- 连接页签控制器抛出的信号
- 接收 `plugin.gd` 下发的统一 model
- 设置页签标题、本地化文本和根级状态灯
- 管理宿主 Dock 页签切换与焦点恢复

---

## 根控制器的数据流

当前 Dock 使用单向数据流：

```text
plugin.gd
  -> _build_dock_model()
  -> mcp_dock.gd.apply_model(model)
  -> server_tab.gd / tools_tab.gd / config_tab.gd
  -> 用户操作 signal
  -> plugin.gd
```

优点：

- UI 层不直接读写 server 或 settings
- 所有页签都以同一份 model 为准
- 语言切换、缩放与状态刷新可以一起重绘

---

## 信号桥接

`mcp_dock.gd` 当前桥接的信号大致分为三类：

### 服务控制

- `port_changed`
- `auto_start_toggled`
- `log_level_changed`
- `permission_level_changed`
- `language_changed`
- `start_requested`
- `restart_requested`
- `stop_requested`
- `full_reload_requested`

### 工具页操作

- `delete_user_tool_requested`
- `tool_toggled`
- `category_toggled`
- `domain_toggled`
- `tree_collapse_changed`

### 配置页操作

- `cli_scope_changed`
- `config_platform_changed`
- `config_write_requested`
- `copy_requested`

这些信号都统一回流到 `plugin.gd`，再由入口调用服务对象。

## 页签实例化方式

`mcp_dock.gd` 在 `_ensure_tabs()` 中：

1. 清空 `TabContainer` 下现有子节点。
2. 依次加载并实例化：
   - `server_panel.tscn`
   - `tools_tab.tscn`
   - `config_panel.tscn`
3. 检查页签控制器是否实现 `apply_model()`。

这样做的直接结果是：

- 各页签可独立维护节点树
- 入口脚本不需要关心页签内部控件
- 后续新增页签只需要新增场景和控制器，再接入根场景

## 缩放与布局策略

UI 缩放仍以 `EditorInterface.get_editor_scale()` 为统一倍率来源，但当前实现区分两类布局参数：

### 运行时仍由脚本控制

- 控件最小高度
- 工具树列宽
- 动态生成卡片内容的最小高度
- 阴影尺寸

### 优先由场景控制

- `MarginContainer` 的边距
- `VBoxContainer/HBoxContainer/GridContainer` 的间距
- 分隔线相对位置
- `VSplitContainer` 的分界布局

这意味着页签脚本只保留“缩放相关硬约束”，常规 UI 间距优先放回 `.tscn` 手动调整。

---

## 当前 UI 目录的维护原则

- 页签控制器只处理本页签的展示和交互
- Dock 根控制器负责跨页签协调与信号转发
- 场景控件命名保持语义清晰，优先使用 `XxxPanel`、`XxxMargin`、`XxxSeparator`、`XxxContent`
- 非必要不在脚本中覆盖普通 margin / separation
