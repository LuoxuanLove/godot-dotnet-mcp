# Godot.NET MCP 文档概述

本文档集用于说明 `addons/godot_dotnet_mcp` 的完整技术实现。当前目录不再只覆盖“如何使用”，而是同时覆盖：

- 插件入口、生命周期、运行时服务与协议路由
- Dock UI 场景、页签脚本、布局与数据流
- 工具系统、工具域划分、配置持久化、本地化与自进化能力
- 目录树、关键文件职责与常见持久化格式

默认读者分为三类：

- 需要在 Godot 编辑器中接入和使用插件的使用者
- 需要继续维护 `plugin.gd`、`plugin/runtime/*`、`ui/*` 与 `tools/*` 的开发者
- 需要快速理解目录结构、文件职责与排障入口的协作者

## 常见工作流

### 1. 启动本地 MCP 服务

1. 在 Godot 编辑器中启用插件。
2. 打开 `MCPDock > 服务器`。
3. 检查端口、自动启动、日志等级和权限等级。
4. 点击“启动”或保持自动启动。
5. 通过 `http://127.0.0.1:3000/mcp` 或当前端口对应地址接入客户端。

对应说明详见 [架构/配置与界面.md](架构/配置与界面.md)、[界面/Server与Config页实现.md](界面/Server与Config页实现.md) 与 [架构/运行时服务.md](架构/运行时服务.md)。

### 2. 调整工具暴露范围与预设

1. 打开 `MCPDock > 工具`。
2. 选择内置 profile 或自定义 profile。
3. 按 `domain -> category -> tool` 三层树结构启停工具。
4. 观察启用数变化，并确认 `disabled_tools` 与当前预设是否一致。
5. 如需共享或回放当前工具集，使用导入 / 导出配置。

对应说明详见 [界面/Tools页实现.md](界面/Tools页实现.md)、[模块/工具系统.md](模块/工具系统.md) 与 [附录/配置与持久化.md](附录/配置与持久化.md)。

### 3. 查看并写入客户端连接配置

1. 打开 `MCPDock > 配置`。
2. 选择目标客户端平台。
3. 核对当前服务地址、生成文本和目标路径。
4. 需要落地到本机时，使用一键写入；只想手动处理时，使用复制。

对应说明详见 [界面/Server与Config页实现.md](界面/Server与Config页实现.md)、[架构/配置与界面.md](架构/配置与界面.md) 与 [附录/配置与持久化.md](附录/配置与持久化.md)。

### 4. 排查 C# 导出绑定与场景问题

1. 用 `script_inspect` 或 `script_exports` 读取脚本导出成员。
2. 用 `scene_bindings` 分析当前场景或指定场景的绑定情况。
3. 用 `scene_audit` 获取结构化问题列表。
4. 必要时配合 `node_query`、`scene_hierarchy` 与 `script_references` 回查节点路径和脚本引用。

对应说明详见 [模块/脚本与场景分析.md](模块/脚本与场景分析.md)、[模块/NET支持.md](模块/NET支持.md) 与 [模块/工具域索引.md](模块/工具域索引.md)。

### 5. 管理 User 工具与审计

1. 用 `plugin_evolution_list_user_tools` 查看当前 User 工具。
2. 用 `plugin_evolution_scaffold_user_tool` 做未授权预演。
3. 确认结果后传 `authorized=true` 执行创建。
4. 用 `plugin_evolution_user_tool_audit` 检查授权与删除记录。
5. 如需在 Dock 中显示 `User` 分类，可配合 `plugin_developer_user_visibility`。

对应说明详见 [模块/自进化系统.md](模块/自进化系统.md)、[模块/核心实现.md](模块/核心实现.md) 与 [模块/工具域索引.md](模块/工具域索引.md)。

## 文档维护约束

- 文档默认以 2026-03-12 之后的重构版本为准，旧版“大 `plugin.gd` 动态拼装 UI”的描述已不再适用。
- 修改工具返回字段、设置键名、profile 文件格式、客户端配置模板或权限边界时，必须同步更新对应章节。
- 修改 UI 场景结构、分界容器、控件命名或页签行为时，必须同步更新 [界面/](界面/) 下相关文档。
- 修改运行时服务、Autoload、重载链路、自诊断记录字段或 server 生命周期时，必须同步更新 [架构/生命周期与装配.md](架构/生命周期与装配.md) 与 [架构/运行时服务.md](架构/运行时服务.md)。
- 新增工具域、迁移 executor 目录、调整兼容 wrapper 或新增 User 工具脚手架能力时，必须同步更新 [模块/工具域索引.md](模块/工具域索引.md) 与 [附录/目录树与文件职责.md](附录/目录树与文件职责.md)。
