# Godot.NET MCP 文档概述

本文档目录用于说明 `addons/godot_dotnet_mcp` 的架构、界面操作、工具调用规范、自进化系统和 Godot.NET 分析能力。文档默认面向两类读者：

- 需要在 Godot 编辑器内使用 Dock、切换工具集、生成 MCP 客户端配置的使用者
- 需要继续维护 `plugin.gd`、`plugin/runtime/mcp_http_server.gd` 与 `tools/*.gd` 的开发者

## 常见工作流

### 1. 启动本地 MCP 服务

1. 在 Godot 编辑器中启用插件。
2. 打开 `MCPDock > 服务器`。
3. 检查端口、自动启动和日志等级。
4. 点击“启动”或保持自动启动。
5. 通过 `http://127.0.0.1:3000/mcp` 或当前端口对应地址接入客户端。

对应说明详见 [架构/配置与界面.md](架构/配置与界面.md) 与 [架构/服务与路由.md](架构/服务与路由.md)。

### 2. 裁剪工具暴露范围

1. 打开 `MCPDock > 工具`。
2. 选择内置 profile 或自定义 profile。
3. 按 domain、category、tool 三层缩进结构开关启停工具。
4. 观察启用数变化，并确认服务端禁用列表已同步。

对应说明详见 [模块/工具系统.md](模块/工具系统.md)。

### 3. 查看连接配置

1. 打开 `MCPDock > 配置`。
2. 查看可直接写入的桌面客户端 JSON 配置。
3. 检查当前服务地址、配置内容和写入目标路径。
4. 如需直接写入本机配置文件，使用一键写入按钮。

补充：

- 语言切换菜单会显示“自称（当前语言译名）”格式，便于在非母语界面下识别目标语言。
- 插件 UI 文本已统一走 localization key；如果界面出现英文回退，优先检查语言资源是否缺 key。

对应说明详见 [架构/配置与界面.md](架构/配置与界面.md)。

### 4. 排查 C# 导出绑定问题

1. 用 `script_inspect` 或 `script_exports` 读取脚本导出成员。
2. 用 `scene_bindings` 分析当前场景或指定场景的绑定情况。
3. 用 `scene_audit` 获取结构化问题列表。
4. 如需回查节点路径，再配合 `scene_hierarchy` 或 `node_query`。

对应说明详见 [模块/脚本与场景分析.md](模块/脚本与场景分析.md) 与 [模块/NET支持.md](模块/NET支持.md)。

### 5. 管理 User 工具

1. 先用 `plugin_evolution_list_user_tools` 查看当前 User 工具列表。
2. 用 `plugin_evolution_scaffold_user_tool` 做未授权预演。
3. 确认结果后再传 `authorized=true` 执行创建。
4. 用 `plugin_evolution_user_tool_audit` 检查审计记录。
5. 如需在 Dock 中显示 `User` 分类，可配合 `plugin_developer_user_visibility`。

对应说明详见 [模块/自进化系统.md](模块/自进化系统.md)。

## 文档使用约束

- 文档默认以 2026-03-10 之后的重构版本为准，旧版大 `plugin.gd` 动态拼装 UI 的描述已不再适用。
- 如果修改了工具返回字段、设置键名、profile 文件格式或客户端配置模板，必须同步更新相应章节。
- 如果修改了界面结构、按钮行为或缩放策略，必须同步更新 [架构/配置与界面.md](架构/配置与界面.md)。
- 如果新增了插件工具分类、User 工具工作流或权限边界，必须同步更新 [模块/工具系统.md](模块/工具系统.md) 与 [模块/自进化系统.md](模块/自进化系统.md)。
