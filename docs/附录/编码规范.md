# 编码规范

本文档约束 `godot-dotnet-mcp` 仓库中的文本文件编码，目标是同时满足：

- 跨平台代码与配置文件的可维护性
- Windows 开发环境下的稳定读取
- 当前 `Windows PowerShell 5.1` 工作流的可操作性

当前策略的核心原则是：

- 默认使用 `UTF-8` 无 BOM
- 仅在明确需要兼容 `Windows PowerShell 5.1` 默认读取行为或 Agent 记忆文件消费时，才使用 `UTF-8 with BOM`

当前规则已经通过以下配置文件落地：

- `godot-dotnet-mcp/.editorconfig`：约束仓库内正式源码、配置、文档、`agent_workflow` 与 `*.ps1`
- `e:/Project/LuoxuanLove/.editorconfig`：仅约束工作区根目录的 `AGENTS.md` 与 `CLAUDE.md`

---

## 总体规则

### 必须使用 UTF-8 无 BOM

以下文件类型和目录默认必须使用 `UTF-8` 无 BOM：

- 所有源码文件：
  - `*.cs`
  - `*.gd`
  - `*.shader`
- 所有工程与配置文件：
  - `*.csproj`
  - `*.props`
  - `*.targets`
  - `*.sln`
  - `*.json`
  - `*.yml`
  - `*.yaml`
  - `*.xml`
  - `*.toml`
  - `*.cfg`
  - `*.editorconfig`
  - `.gitignore`
  - `.gitattributes`
- 所有 Godot 文本资源：
  - `*.tscn`
  - `*.tres`
- 所有正式对外文档：
  - `README*.md`
  - `docs/**/*.md`

原因：

- 这是跨平台工具链最稳定、最通用的默认编码。
- 可避免 BOM 干扰脚本头、配置解析器和部分严格工具。
- 与当前 `Central Server` / `.NET` / Godot 文本资源的主流约定一致。

### 应使用 UTF-8 with BOM

以下文件类型和目录在本仓库中应使用 `UTF-8 with BOM`：

- `AGENTS.md`
- `CLAUDE.md`
- `agent_workflow/**/*.md`
- `*.ps1`

原因：

- 这些文件在当前 Windows 开发环境中，经常被 `Windows PowerShell 5.1` 直接读取或执行。
- `PowerShell 5.1` 默认读取无 BOM 的 UTF-8 中文文件时容易乱码。
- Agent 记忆文件与本地工作流文档需要保证在默认 Windows 工具链中直接打开即可正确显示。

## PowerShell 5.1 约束

当前环境仍包含 `Windows PowerShell 5.1`，因此必须遵守以下规则：

### 读取 UTF-8 文件

- 读取无 BOM UTF-8 文件时，必须显式指定编码。
- 禁止依赖默认 `Get-Content` 去读取正式源码、配置和 `docs/**/*.md` 中的中文内容。

推荐写法：

```powershell
Get-Content -Encoding UTF8 -Path .\docs\附录\编码规范.md
```

### 写入 UTF-8 文件

- 禁止依赖默认 `Set-Content` / `Out-File` 写入仓库文件，因为默认行为可能落成 ANSI 或不符合目标编码。
- 写入必须显式指定目标编码。
- 需要写入无 BOM UTF-8 时，不要在 `PowerShell 5.1` 中依赖 `Set-Content -Encoding UTF8`，因为它会写出 BOM。

推荐策略：

- 正式源码、配置、对外文档：
  - 优先使用编辑器保存为 `UTF-8` 无 BOM
  - 或使用 `.NET` API 明确写入无 BOM
- `AGENTS.md`、`CLAUDE.md`、`agent_workflow/**/*.md`、`*.ps1`：
  - 明确写入 `UTF-8 with BOM`

无 BOM 写入示例：

```powershell
[System.IO.File]::WriteAllText(
    $path,
    $content,
    (New-Object System.Text.UTF8Encoding($false))
)
```

带 BOM 写入示例：

```powershell
[System.IO.File]::WriteAllText(
    $path,
    $content,
    (New-Object System.Text.UTF8Encoding($true))
)
```

## 执行规则

### 新建文件

- 新建源码、配置、正式文档时，默认创建为 `UTF-8` 无 BOM。
- 新建 `*.ps1`、`AGENTS.md`、`CLAUDE.md`、`agent_workflow/**/*.md` 时，默认创建为 `UTF-8 with BOM`。

### 修改现有文件

- 修改文件时优先保持其所属类别的目标编码，不混用。
- 如果发现现有文件编码与本规范冲突，在不影响当前任务的前提下应顺手收口到目标编码。
- 修改编码规则时，必须同步更新 `.editorconfig` 与本文档，不能只改其中一处。

### 审查与重构时

- 遇到中文乱码，先检查是否是 `PowerShell 5.1` 默认读取无 BOM UTF-8 导致，不要先假设文件已损坏。
- 评估“仓库是否干净”时，不只看内容，还要看编码是否符合本规范。

---

## 例外说明

- 如果未来仓库主开发环境切换到 `PowerShell 7`，本规范可以重新评估，并考虑进一步减少 BOM 文件范围。
- 如果某个第三方工具明确要求特殊编码，以该工具的硬约束为准，但必须在对应文档中说明原因。
