# 配置与持久化

本文档汇总插件当前使用的配置路径、持久化文件格式和导入导出约束。

---

## 持久化路径

| 路径 | 类型 | 作用 |
|---|---|---|
| `user://godot_dotnet_mcp_settings.json` | JSON | 插件设置 |
| `user://godot_dotnet_mcp_tool_profiles/*.json` | JSON | 自定义工具 profile |
| 用户指定路径 | JSON | `plugin_developer` 导入 / 导出工具配置 |

---

## 插件设置文件

路径：

```text
user://godot_dotnet_mcp_settings.json
```

当前主要字段：

| 字段 | 说明 |
|---|---|
| `port` | 内嵌 server 端口 |
| `host` | 监听地址 |
| `auto_start` | 是否自动启动 |
| `debug_mode` | 是否输出调试日志 |
| `log_level` | 日志等级 |
| `permission_level` | 当前 permission level |
| `disabled_tools` | 禁用工具全名列表 |
| `tool_profile_id` | 当前选中 profile |
| `language` | 当前语言 |
| `show_user_tools` | 是否在 Tools 页显示 User 分类 |
| `collapsed_categories` | 当前折叠的 category 列表 |
| `collapsed_domains` | 当前折叠的 domain 列表 |
| `collapsed_system_tools` | Tools 页中默认折叠的 系统工具名列表（默认包含全部 15 个内置工具名） |

缺失字段会在加载时用 `PluginSettingsSchema.DEFAULT_SETTINGS` 补齐。

---

## 自定义 Profile 文件

路径：

```text
user://godot_dotnet_mcp_tool_profiles/<slug>.json
```

当前格式：

```json
{
	"name": "My Profile",
	"disabled_tools": [
		"scene_scene_run",
		"plugin_evolution_plugin_evolution_delete_user_tool"
	]
}
```

说明：

- 文件名不直接等于 profile id，profile id 为 `custom:<slug>`
- `disabled_tools` 必须是字符串数组
- 重命名 profile 时会新写文件，再删除旧文件

---

## 工具配置导入 / 导出文件

由 `plugin_developer_export_config` / `plugin_developer_import_config` 使用。

当前格式：

```json
{
	"format_version": 1,
	"profile_id": "default",
	"disabled_tools": [
		"scene_scene_run"
	]
}
```

校验规则：

- `profile_id` 不能为空
- `disabled_tools` 必须是字符串数组
- 数组内元素不能为空字符串

导入成功后不会直接改变文件格式，只回读为结构化数据再交给插件状态。

---

## 客户端配置写入策略

桌面端客户端配置写入由 `ClientConfigService.write_config_file()` 实现。

策略如下：

1. 先解析当前 UI 中展示的 JSON 文本。
2. 若目标文件已存在，尝试读取并解析原 JSON。
3. 只操作 `mcpServers` 节点。
4. 只更新 `godot-mcp` 对应项，不清空其它 server。
5. 目录不存在时先递归创建。

这意味着插件写入是“最小合并”，不是“全文件覆盖模板”。

---

## CLI 配置

CLI 客户端当前不做直接写盘，而是生成命令文本，例如：

- Claude Code
- Codex

其中 Claude Code 还区分：

- `user`
- `project`

作用域切换只影响生成的命令文本，不影响其它设置。

---

## 客户端安装检测缓存

客户端安装检测由 `ClientInstallDetectionService` 在内存中维护短期缓存：

- 缓存 TTL：`5000ms`
- 缓存内容：客户端 executable path、config entry 状态、runtime running 状态
- 缓存不会写入 `user://`

当前缓存失效入口：

- Config 页手动选择 / 清除客户端路径
- Config 写入 / 删除完成后
- Dock 切换到 Config 页时

这意味着客户端检测属于运行期投影状态，而不是持久化配置本身。

---

## 本地化资源

语言资源并不写入 `user://`，而是直接来自：

- `localization/locale_*.gd`

当前本地化切换策略是：

- 设置文件里记录用户选择的语言
- 插件启动时恢复该语言
- 所有 UI 文本由 `LocalizationService` 统一取值

---

## 运行时桥持久化相关约束

运行时桥通过 Autoload 方式挂入项目：

- 名称：`MCPRuntimeBridge`
- 路径：`res://addons/godot_dotnet_mcp/plugin/runtime/mcp_runtime_bridge.gd`

这类配置写入的是 Godot 项目设置，而不是插件自己的 `user://` JSON。

---

## 常见排障点

- 改了场景中的 margin 但不生效：先确认对应页签脚本是否仍在运行时覆盖该属性。
- profile 导入失败：优先检查 `profile_id` 是否为空、`disabled_tools` 是否是字符串数组。
- 一键写入客户端失败：优先检查目标目录是否可写、现有 JSON 是否能解析。
- 语言切换后有 key 原样显示：优先检查目标 `locale_*.gd` 是否缺键。
